## Домашнее задание по теме Микросервисы - Масштабирование ##  

### Задание 1 - Кластеризация ###  
Задачу по предъявляемым требованиям в принципе можно решить как с помощью Docker swarm, так и и с помощью Kubernetes.  
В Kubernetes из коробки присутствуют встроенные механизмы для:  
- разграничения доступа к сервисам
- распределения нагрузки между приложениями
- автоматического горизонтального (HPA), вертикального (VPA), управляемого событиями (KEDA) масштабирования
- хренения чувствительных данных Secret

В swarm поставленные задачи решаются сложнее либо с применением дополнительных продуктов, в частности нет автоматического масштабирования.
  
В случае необходимости для решения отдельных задач к Kubernetes можно добавить продукты:  
  
- Apache ApiSix - для маршрутизации сервисов
- Hashicorp Consul - для хранения параметров сервисов и реализации Service Registry
- Hashicorp Vault/Openbao/Orion StarVault - для безопасного хранения секретов

Решение RedHat Openshift построено на Kubernetes и является надстройкой его надстройкой и развитием. 
Продукт коммерческий, платный. В Openshift присутствуют встроенные инструменты CI/CD. 
  
Учитывая то, что Openshift - коммерческий продукт, а его владелец IBM в числе первых покинула Россию в 2022 году, Kubernetes выглядит наиболле подходящим, с точки зрения начинающего DevOps инженера, который еще не изучал тему Kubernetes.
Недостающие встроенные инструменты интеграции и развертывания могут быть заменены, например GitLab.
